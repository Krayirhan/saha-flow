## Sahiplik Zinciri

| Katman | Sorumlu Rol | Sorumluluk Alanı |
|---|---|---|
| İş Problemi → Ürün Gereksinimi | Product Owner | Problemin doğru tanımlandığı, gereksinimin problemi çözdüğü |
| Ürün Gereksinimi → Kullanıcı Hikâyesi | Product Owner + Tech Lead | Hikâyelerin gereksinimi karşıladığı, kabul kriterlerinin ölçülebilir olduğu |
| Kullanıcı Hikâyesi → Mimari Karar | Tech Lead + Backend/Frontend Lead | Mimarinin hikâyeyi desteklediği, teknik borç oluşturmadığı |
| Mimari Karar → Güvenlik Kontrolü | Tech Lead + Güvenlik (V1'de harici danışman) | Güvenlik kontrollerinin ASVS L2'yi karşıladığı |
| Güvenlik Kontrolü → Test | QA / Geliştirici (ekipler arası test) | Testlerin güvenlik kontrollerini doğruladığı |
| Test → Telemetry/Metrik | DevOps / Backend Lead | Metriklerin canlıda çalıştığı, alarm eşiklerinin doğru olduğu |
| Tüm Zincir | Product Owner (nihai sorumlu) | Uçtan uca izlenebilirlik ve kalite |

## Karar Bekleyen Konular

1. Güvenlik kontrolleri için harici danışman MVP'de gerekli mi, yoksa V1'de mi alınmalı? → V1'de.
2. KVKK uyumluluğu için harici hukuk danışmanı ne zaman devreye girmeli? → Pilot aşamasının başında (W14).
3. Telemetry backend'i: Grafana Cloud mu, self-hosted Grafana mı? → MVP'de self-hosted, V1'de managed.
4. E2E test otomasyonu kapsamı: MVP'de sadece kritik akış mı, tüm akışlar mı? → Kritik akış.
5. ADR numaralandırması: her ADR'ye statik ID verilmeli, bu matriste referans ADR-00X kullanılmıştır; gerçek ADR oluşturuldukça ID'ler güncellenmelidir.
